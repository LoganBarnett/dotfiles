#+author:    System Documentation
#+date:      2026-02-14

* Overview

This setup enables automatic code review across all your projects using
Claude's agent system.  Your review agents are globally available and can be
used in any repository.

* Quick Start

** For a New Project

#+begin_src bash
cd /path/to/your/project
~/dev/dotfiles/nix/claude-template/setup-claude-project.sh
#+end_src

This creates a ~CLAUDE.md~ file that instructs Claude to automatically run
code reviews.

** For an Existing Project

Just copy the template:

#+begin_src bash
cp ~/dev/dotfiles/nix/claude-template/CLAUDE.md /path/to/project/
#+end_src

* How It Works

** Global Agents

Your agents live in ~$HOME/.claude/agents/~ and are available everywhere:
- ~standards-reviewer~: Checks code standards compliance.
- ~senior-engineer~: Reviews engineering best practices.
- ~code-review-orchestrator~: Coordinates reviews.
- ~work-mode~: Handles tasks with automatic review.

** Per-Project Instructions

Each project has a ~CLAUDE.md~ file that tells Claude to use these agents
automatically.  When you ask Claude to work on code, it will:

1. Make the requested changes.
2. Invoke review agents automatically.
3. Fix critical issues.
4. Report results.

* Working Across Projects

** Method 1: Let Claude Handle It

Just tell Claude which project you're working on:

#+begin_example
"I'm working on my-app project.  Add a new user authentication module."
#+end_example

Claude will:
- Work in that project's directory.
- Follow that project's CLAUDE.md.
- Use your global agents for review.

** Method 2: Be Explicit About Context

#+begin_example
"In the ~/projects/web-app directory, refactor the database module to use
connection pooling."
#+end_example

** Method 3: Use Work Mode Agent

#+begin_example
"Use work-mode agent to implement feature X in ~/projects/api-server"
#+end_example

* Customizing Per Project

Each project's ~CLAUDE.md~ can have additional rules:

#+begin_src org
* Project-Specific Guidelines

** Python Standards
- Use type hints for all functions.
- Follow PEP 8 style guide.

** Testing Requirements
- Minimum 80% test coverage.
- All endpoints must have integration tests.
#+end_src

* Advanced: Project Types

Create specialized templates:

#+begin_src bash
# Python project template
cp claude-template/CLAUDE-python.md ~/projects/my-python-app/CLAUDE.md

# Rust project template
cp claude-template/CLAUDE-rust.md ~/projects/my-rust-lib/CLAUDE.md
#+end_src

* Your Friend's Approach

Your friend likely:
1. Has agents defined globally (like you do now).
2. Uses ~cd~ to change to different project directories.
3. Has CLAUDE.md in each project with specific rules.

This is exactly what your setup now supports!

* Tips

1. **Global agents, local rules**: Agents are global, but each project's
   CLAUDE.md contains project-specific standards.

2. **Review history**: Each project gets its own ~reviews/~ directory for
   tracking review history.

3. **Skip when needed**: Tell Claude "skip reviews" when you need quick
   experiments.

4. **Project awareness**: Claude reads CLAUDE.md from the current working
   directory automatically.
